Create Projects

Create a new requirement engineering project. Make you project public, discoverable or private. A public project is open for anyone in the community to subscribe and start contributing. A discoverable project is a semi-public project where people within the community can discover the project and request membership. The project administrator can accept or deny the request. Private project is not discoverable to the community. Only administrator can invite members.

Organize your project

Organize your requirement project in viewpoints, goals, requirements, scenarios, processes and use cases.

Viewpoints

Describes possible diverse sources of requirements. They provide high level visualization of a system and mechanisms for identification, structuring, organization and classification of systems requirements.

Goals

A goal is defined as a final decomposed desired outcome to be achieved or satisfied by a composite system. Goals are therefore identified and formulated in hierarchies using goal graph at different levels of abstraction from highest strategic level to lowest levels of operation.

Requirements

is a description of features and functionalities of the target system. Requirements convey the expectations of users from the software product.

Scenario

A scenario is a scene that illustrates some interaction with a proposed system. A scenario is a tool used during requirements analysis to describe a specific use of a proposed system. Scenarios capture the system, as viewed from the outside, e.g, by a user, using specific examples.

Process

Is a set of connected activities performed in a defined order that will accomplish a specific organizational goal. It is a step by step procedures of doing something.

Use case

Is a list of actions or event steps typically defining the interactions between a role and a system to achieve a goal. The actor can be a human or other external system.

Invite Members

If a project is Private Project Admin (Creator) can Invite members to Join the Project for contributions.

Membership Requests

If the Project is Private and Discoverable, system users can ask to join a Project, Project admin can accept or Reject a request.

Project Members

Project admin can view Project Members details, Suspend,even remove them from the project when necessary.

Project Incentives

Project Creator can provide Incentives to Contributors of his/her project. this gift can be monetary gift or other things.
